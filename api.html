chat_with_tools(messages, tools=None, tool_params=None, model=None, max_tool_calls=5, files=[])[source]

Send a chat completion request and handle tool calls automatically.

This will automatically execute tool calls and include their results in subsequent API calls until the model returns a final response.

Parameters:

  • messages (List[ChatCompletionMessageParam]) – List of message dictionaries with ‘role’ and ‘content’ keys

  • tools (Optional[Sequence[str]]) – List of tool names to use (None for all registered tools)

  • tool_params (Optional[Dict[str, Dict[str, Any]]]) – Optional parameters to pass to the tools when they are called

  • model (Optional[str]) – Model to use (defaults to the client’s default model)

  • max_tool_calls (int) – Maximum number of tool call rounds to allow

  • files (Iterable[Path]) – Optional list of Path objects to files that should be included with the request

Return type:

str

Returns:

The final assistant message content (str) after all tool calls are processed

Raises:

RuntimeError – If the maximum number of tool calls is exceeded

Example

>>> client = OpenWebUIClient()
>>> client.tool_registry.register(get_weather)
>>> response = client.chat_with_tools(
...     messages=[{"role": "user", "content": "What's the weather in Paris?"}],
...     max_tool_calls=3
... )
>>> print(response)

Create a chat completion with support for the ‘files’ parameter.

This overrides the standard create method to handle the ‘files’ parameter that OpenWebUI supports but is not in the standard OpenAI API.

Parameters:

  • messages (Iterable[ChatCompletionMessageParam]) – A list of messages comprising the conversation so far.

  • model (Union[str, ChatModel]) – ID of the model to use.

  • files (Union[Collection[FileObject], None, NotGiven]) – A list of file IDs to attach to the completion request (OpenWebUI specific).

  • parameters (# Standard OpenAI)

  • details (see OpenAI API docs for)

  • audio (Union[ChatCompletionAudioParam, None, NotGiven]) – Audio input parameters.

  • frequency_penalty (Union[float, None, NotGiven]) – Penalizes repeated tokens according to frequency.

  • function_call (Union[FunctionCall, NotGiven]) – Controls how the model uses functions.

  • functions (Union[Iterable[Function], NotGiven]) – Functions the model may call to interact with external systems.

  • logit_bias (Union[Dict[str, int], None, NotGiven]) – Modifies likelihood of specific tokens appearing in completion.

  • logprobs (Union[bool, None, NotGiven]) – Whether to return log probabilities of the output tokens.

  • max_completion_tokens (Union[int, None, NotGiven]) – Maximum number of tokens that can be generated for completions.

  • max_tokens (Union[int, None, NotGiven]) – Maximum number of tokens to generate in the response.

  • metadata (Union[Metadata, None, NotGiven]) – Additional metadata to include in the completion.

  • modalities (Union[List[Literal['text', 'audio']], None, NotGiven]) – List of modalities the model should handle.

  • n (Union[int, None, NotGiven]) – How many completions to generate for each prompt.

  • parallel_tool_calls (Union[bool, NotGiven]) – Whether function and tool calls should be made in parallel.

  • prediction (Union[ChatCompletionPredictionContentParam, None, NotGiven]) – Control specifics of prediction content.

  • presence_penalty (Union[float, None, NotGiven]) – Penalizes new tokens based on their presence so far.

  • reasoning_effort (Union[ReasoningEffort, None, NotGiven]) – Controls how much effort the model spends reasoning.

  • response_format (Union[ResponseFormat, NotGiven]) – Format in which the model should generate responses.

  • seed (Union[int, None, NotGiven]) – Enables deterministic sampling for consistent outputs.

  • service_tier (Union[Literal['auto', 'default', 'flex'], None, NotGiven]) – The service tier to use for the request.

  • stop (Union[str, None, List[str], NotGiven]) – Sequences where the API will stop generating further tokens.

  • store (Union[bool, None, NotGiven]) – Whether to persist completion for future retrieval.

  • stream (Union[Literal[False], None, Literal[True], NotGiven]) – Whether to stream back partial progress.

  • stream_options (Union[ChatCompletionStreamOptionsParam, None, NotGiven]) – Options for streaming responses.

  • temperature (Union[float, None, NotGiven]) – Controls randomness in the response.

  • tool_choice (Union[ChatCompletionToolChoiceOptionParam, NotGiven]) – Controls how the model selects tools.

  • tools (Union[Iterable[ChatCompletionToolParam], NotGiven]) – List of tools the model may call.

  • top_logprobs (Union[int, None, NotGiven]) – Number of log probabilities to return per token.

  • top_p (Union[float, None, NotGiven]) – Controls diversity via nucleus sampling.

  • user (Union[str, NotGiven]) – Unique identifier representing your end-user.

  • web_search_options (Union[WebSearchOptions, NotGiven]) – Options to configure web search behavior.

  • requests (# Additional parameters for HTTP)

  • extra_headers (Optional[Headers]) – Additional HTTP headers.

  • extra_query (Optional[Query]) – Additional query parameters.

  • extra_body (Optional[Body]) – Additional body parameters.

  • timeout (Union[float, Timeout, None, NotGiven]) – Request timeout in seconds.

Return type:

Union[ChatCompletion, ChatCompletionChunk]

Returns:

A ChatCompletion object containing the model’s response.
